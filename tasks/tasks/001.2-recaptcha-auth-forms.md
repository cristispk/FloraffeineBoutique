# Task 001.2 — Add Google reCAPTCHA v2 to Auth Forms

## Goal

Add Google reCAPTCHA v2 checkbox protection to the user and merchant authentication forms, with server-side verification in Laravel, Romanian user-facing messages, and no behavioral changes beyond requiring a valid captcha.

---

## Scope

### Forms to protect

#### Public user area:
- resources/views/public/auth/register.blade.php
- resources/views/public/auth/login.blade.php

#### Merchant area:
- resources/views/merchant/auth/register.blade.php
- resources/views/merchant/auth/login.blade.php

---

### Controllers impacted

#### User:
- App\Http\Controllers\User\Auth\RegisterController
- App\Http\Controllers\User\Auth\LoginController

#### Merchant:
- App\Http\Controllers\Merchant\Auth\RegisterController
- App\Http\Controllers\Merchant\Auth\LoginController

---

## Requirements

### Configuration

Add environment variables to `.env`:

```env
RECAPTCHA_ENABLED=true
RECAPTCHA_SITE_KEY=
RECAPTCHA_SECRET_KEY=
```

- Use RecaptchaService inside Form Requests for validation
- Do not implement captcha logic directly in controllers
- Use config/recaptcha.php for all configuration access
- If RECAPTCHA_ENABLED=false, validation and UI must skip reCAPTCHA entirely

---

## Implementation Notes (Completed)

- `config/recaptcha.php` centralizes configuration:
  - `enabled` (`RECAPTCHA_ENABLED`)
  - `site_key` (`RECAPTCHA_SITE_KEY`)
  - `secret_key` (`RECAPTCHA_SECRET_KEY`)
- `RecaptchaService` in `app/Services/Security/RecaptchaService.php`:
  - `isEnabled()` depends only on `config('recaptcha.enabled')`
  - `verify($token, $ip)`:
    - returns `true` when reCAPTCHA is disabled
    - returns `false` if enabled but keys are missing, the token is empty, or verification fails
- Validation is implemented in Form Requests only:
  - `UserRegisterRequest`, `UserLoginRequest`
  - `MerchantRegisterRequest`, `MerchantLoginRequest`
  - When reCAPTCHA is enabled:
    - missing token → "Te rugăm să confirmi că nu ești robot."
    - invalid/failed verification → "Verificarea reCAPTCHA a eșuat. Te rugăm să încerci din nou."
- Blade views for protected forms:
  - `resources/views/public/auth/register.blade.php`
  - `resources/views/public/auth/login.blade.php`
  - `resources/views/merchant/auth/register.blade.php`
  - `resources/views/merchant/auth/login.blade.php`
  - Each renders the reCAPTCHA widget and Google script only when `config('recaptcha.enabled')` is `true`.
  - No other behavior or routes were changed.
